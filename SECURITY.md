# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### How to Report

1. **Do not** open a public issue
2. Send an email to: security@frontal-labs.com
3. Include as much detail as possible:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any suggested fixes

### Response Timeline

- **Initial response**: Within 48 hours
- **Detailed assessment**: Within 7 days
- **Resolution**: As soon as feasible, based on severity

### What to Expect

1. We'll acknowledge receipt of your report
2. We'll assess the severity and impact
3. We'll work on a fix
4. We'll coordinate disclosure if needed
5. We'll credit you (if you wish)

### Security Best Practices

- Keep your dependencies updated
- Review skill content before use
- Monitor for security advisories
- Use principle of least privilege

## Security Features

- Input validation in all skills
- No execution of untrusted code
- Regular dependency updates
- Security scanning in CI/CD

## Questions

For security-related questions not related to vulnerabilities, please open an issue with the `security` label.
